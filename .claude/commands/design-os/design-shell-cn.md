# 设计界面外壳 (Design Shell)

您正在协助顾客去筹划组建其应用程序外衣壳套 (application shell) —— 这即是包裹罩在各大小版块之外那常青持久的界面导航以及排布格局。这项属于界面设计范畴，而非最终实现的运作代码。

## 第 1 步：查实检查先决条件

首要是核定一些需具备之前提文件是不是齐备：

1. 查阅 `/product/product-overview.md` — 获晓产品名头及大致其貌
2. 查阅 `/product/product-roadmap.md` — 摸清用来列进导航条目的分属版块等
3. 检查下 `/product/design-system/colors.json` 和 `/product/design-system/typography.json` 两文件的存续

如若没有 Overview 或 roadmap 其中之一：

"在能够着手设计此款系统的大外层保护衣壳前，请阁下务要先定好自身的产品及辖下界分板块。去运行这：
1. `/product-vision` — 筹算敲实您的整副产品盘算
2. `/product-roadmap` — 规划您的业务分属版块"

但缺一个便停住脚歇在此处。

假使欠缺了那类系统统一样式的通用设定词令（design tokens），可给出个不打紧的温馨提醒但活照干：

"提示：您的专用设计统调标识符等项尚未圈定。我权且拿个原本就有的素淡系统内建缺省格调来操刀应付着先了。但还是更建议先去运行下一趟 `/design-tokens` 为您的全盘风格调性立个长久不改的章法。"

## 第 2 步：研辨产品体态面貌 (Analyze Product Structure)

审看那几块线路图里的分划板块并去提交上供那导航选配的方案：

"现下我正是要替 **[产品其名]** 来做个皮套外壳设计划策。对照着您那路标规划图列着的清单，当下有 [N] 块领地：

1. **[板块 1]** — [详情解说]
2. **[板块 2]** — [详情解说]
3. **[板块 3]** — [详情解说]

咱们现在来拍板它的套壳架构排阵形吧。以下罗列有数种风行的经典款式：

**A. 侧摆侧边导航板 (Sidebar Navigation)** — 摆左道垂行竖立，把内容正席居右留出
   所宜适用：块目名目繁杂之软件，像表板台这类控制器具流，乃至幕后管理的诸般控制面台

**B. 顶着天幕拉开的导航条 (Top Navigation)** — 横摆在高处，让下面内容依次由上往下落
   所宜适用：轻省型应用，宣推流类的货品，以及少数板块者

**C. 极减主义眉眼框 (Minimal Header)** — 不含导航，单挂品牌 Logo 加带个客人自身相关的调派小界面，至于业务跳转那各有它路出。
   所宜适用：干单项职门类活的小把戏工具，流程式导引类单道引水向流程

何种款式更加贴对 **[产品其名]** 此身量？"

等其开尊口落锤敲定。

## 第 3 步：采集细节的图景用料

动用 AskUserQuestion 以去搞清：

- "这有关客官操作用以改像登退等的菜单 (User menu) 阁下以为摆哪妥善？"
- "您想不想在这移动屏幕里把边这侧边板条缩合折叠，还是干脆将其化身作一汉堡形按钮？"
- "这当中可要另添附带什么导引条目么？（似诸如什么系统设置呀，疑惑答问求救援呀 等）"
- "应用起驾发车的一瞬最先该显什么当作家（Home）？"

## 第 4 步：向之奉上这成形之外壳制案大纲

到了摸透摸清它所有喜好的那个点上：

"此即为为 **[产品其名]** 所量身缔作的外衣大壳设计概览谱：

**布局形态结构：** [侧立板/天台边/极减减派]

**引手导航架势构架：**
- [项目条 1] → [跳转领地版块]
- [项目条 2] → [跳转领地版块]
- [项目条 3] → [跳转领地版块]
- [附着多加进来的类于如设定期或援助区]

**关乎于操用客体之用菜单：**
- 落点地：[顶右偏角 / 顺板最下落底]
- 给内含有：如其面容首像，字名号，退出按扣离去键

**给各身姿设备的自适变动态效应对策：**
- 大铺面台式桌面下：[展示貌]
- 执手的小端移交动中：[屈伸变化法]

这和您心境图划所契合上了否？"

交头互动直到对方认可点首放关。
