# 设计标记 (Design Tokens)

您正在协助顾客去筛选设定好他们自家产品将用的颜色与字体设定。这些个设计标记等项全且将得在所有界面设计图及外皮包裹壳上统一地大面积加以落实贯彻运用。

## 第 1 步：查验前置条件

首先去认证检查总概览文件是否存有：

查阅文件 `/product/product-overview.md` 以获悉本产品的意旨内涵所在。

如果不曾有此文件：

"在能够动手设定您的设计体系之前，阁下得先立起建好产品愿景。有劳先敲下跑一趟 `/product-vision` 指令行。"

前置若是不备则停步在此。

## 第 2 步：叙清这流程解这原委

"且让咱们为 **[产品其名]** 来定下它的视觉名牌特征身份吧。

我将在一旁襄助您选取挑下：
1. **各等色彩** — 主打用主色、次要从伴色乃至素净中性等大底子色板
2. **诸各字样** — 作大标排头的，书落正文的一般文体，用来码印代码用的

这些将被始终如一在全身内外各处界面设计连同大外壳体上方方面面贯彻落实着用。

阁下心中可曾有个已成形存有的大致所属品牌基色抑或某式字样有了个底没有？还是说要不要我来参谋几个点子？"

就这般等其主顾的作答回应。

## 第 3 步：定下颜料的选调

打那内含带自有的 Tailwind 现成的色盘子去挑着去帮客定音落色。循着产品的调性种别拿捏出可选项：

"既然在颜色上我们想让其在后续造屏幕成设计里显得毫不违和天然顺畅，那就在 Tailwind 它的一众色板池中来挑用。

**主打发光重点色 (Primary color)** (做主显的强调之光、控件钮、及链入指路用的):
流俗常落的选调：`blue`, `indigo`, `violet`, `emerald`, `teal`, `amber`, `rose`, `lime`

**相佐从配伴角色 (Secondary color)** (做那副的贴签标记花束引注意的地方):
要能和主打的那边互有补佐且呼应的好颜调——常以另一不一样的色相系又或者选一同宗的调但走中性灰路子变着

**底子素中平之色 (Neutral color)** (底布设垫的、排文下笔的及打边围线拉栏网用):
这可选有：`slate` (清冷系的灰), `gray` (干纯的灰), `zinc` (稍带这温暖和感), `neutral`, `stone` (发这石头暖质地的灰)

依照这 **[产品其名]** 之气性，我在下建言：
- **第一主列打主力色应配：** [点拨色子] — [引这就和那配理由原因]
- **用来作副着重第二级佐色应选：** [点拨色子] — [引它这能当补配的个中缘由]
- **给用上个平常素打底的：** [点拨色子] — [说说这用去好的所在原因]

这何种方配对于阁下这产品才觉对了味感觉对了？"

要是其并没有个大概的底线主张可以使用 AskUserQuestion 来向之讨教：

- "总的大格调气场打算怎么的求怎么走？正统老成还是活跳生脱？现代理派还是简平减没致？"
- "有啥颜色是铁定不准绝不可犯绝不可沾上身的？"
- "日头下亮妆态，或者是夜下暗面底妆呢，又还是来全乎的都要兼收并包的？"

## 第 4 步：在诸位字辈型表里定个拔高

来协助去相助其在 Google Fonts 谷字榜里拔选：

"至于字体字形上，咱们直接往那 Google Fonts 门上摘字这就为了之后能利索利落给这上了网络大用。

**写在那门额头头字（Heading font）** (正冠标题、各分头等)：
大多大众偏爱之属：`DM Sans`, `Inter`, `Poppins`, `Manrope`, `Space Grotesk`, `Outfit`

**落笔在这实身（Body font）** (作分段文章叙的、一般性界面常展字样用)：
往往跟头字是相承一宗同一路的，又抑或是可另点诸如这几样：`Inter`, `Source Sans 3`, `Nunito Sans`, `Open Sans`

**打在那专工刻意之处用这单一宽度字体（Mono font）** (充这书写代码的，及诸般技术硬专性文本字之流的)：
可选选项单有：`IBM Plex Mono`, `JetBrains Mono`, `Fira Code`, `Source Code Pro`

拿在这这 **[产品得名]** 上的我会来下这参提见议：
- **在那打字头题上：** [点子选议] — [因啥由于]
- **放在那身为正主体上：** [点子选议] — [因啥由于]
- **放这用在特技专道等同等这等字上头上的：** [点子选议] — [因啥由于]

君意下以为何如？"

## 第 5 步：端陈呈列这最后的收局合拍大作

一旦其这拍板之后：

"那瞧这套这就成阁下之系统色标范设大典系统之基石范：

**着色的这大方向：**
- 出主当先色：`[某种颜]`
- 做辅佐从次色：`[某种颜]`
- 铺个平铺打实基色：`[某种颜]`

**所钦之点字诸型：**
- 拿做高高标题的：[诸字一字名号]
- 作正实里充肉身文本的：[诸字一字名号]
- 单拿作敲行字打代码码等用的实宽度单宽字：[诸字一字名号]

这且入眼瞧着何如好还顺么吗？这就预就都这就起备要将这也存备留进底入存档保存否可？"

## 第 6 步：开筑文件 (Create the Files)

准可过这过关之后那就立刻去建立这二等文件：

**文件 1:** `/product/design-system/colors.json`
```json
{
  "primary": "[color]",
  "secondary": "[color]",
  "neutral": "[color]"
}
```

**文件 2:** `/product/design-system/typography.json`
```json
{
  "heading": "[Font Name]",
  "body": "[Font Name]",
  "mono": "[Font Name]"
}
```

## 第 7 步：完工报功事就宣告收尾 (Confirm Completion)

向用户知告交这一说个差落：

"我都通将其等这也给悉保存留入其档去了：
- `/product/design-system/colors.json`
- `/product/design-system/typography.json`

这在日后将用于制造你的所有下辖诸个别路分块版面的其显示面画面建树工程中去派用场。

接下来可去跑着之的一着步这之一这就下的：将一可就这一趟跑个起唤个唤口令声名为发 `/design-shell`  令其用这就这也这就使这就使就这就去来做你应用程序那一在最上在这包裹在外边的大外衣套包裹层"

## 参考信息

Available colors (each has shades 50-950):
- **Warm:** `red`, `orange`, `amber`, `yellow`, `lime`
- **Cool:** `green`, `emerald`, `teal`, `cyan`, `sky`, `blue`
- **Purple:** `indigo`, `violet`, `purple`, `fuchsia`, `pink`, `rose`
- **Neutral:** `slate`, `gray`, `zinc`, `neutral`, `stone`

## 重要事项注意事项 (Important Notes)

- 颜色应该是 Tailwind 原生配置的英文名
- 字体应当是精确的 Google Fonts 上的英文字名
- 给顾客出点子建议时候务必考虑其产品特性的属性气场
- 代码字体（mono font）是选填项，但如果有代码类产出则极度推荐使用
- Design tokens 的设置只限于产出的业务页面（screen designs），对大全局 Design OS 应用的样式不形成干连。
