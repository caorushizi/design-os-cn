## 你的想法与代码库之间缺失的设计过程。

[Design OS](https://buildermethods.com/design-os) 是一个产品规划和设计工具，它可以帮助您定义产品愿景，勾勒数据形状（data shape），设计 UI，并导出可用于生产环境（production-ready）的组件以供实现。无需直接开始编码，您可以通过一个有向导的流程来确定您要构建什么以及为什么构建，然后将所有内容移交给您的编码 Agent 以正确构建它。

## 问题所在

AI 编码工具在快速构建方面表现出色。但结果往往不尽如人意。您描述了您想要的，Agent 构建了*一些东西*，但那不是您设想的。UI 看起来很普通。功能只实现了一半。您花在修复和重定向上的时间甚至与直接构建所花的时间一样多。

**核心问题：** 我们要求编码 Agent在*同时*弄清楚要构建什么以及去构建它。设计决策是在运行中做出的，深埋在代码中，如果不重新开始，就不可能调整。没有规范（spec）。没有共识。没有什么是“完成”的单一事实来源。

## Design OS 流程

Design OS 提供并支持一个引导式的设计和架构流程。您与 AI 合作，通过结构化的步骤逐步完成：

1. **产品规划** — 定义您的愿景，分解您的路线图，并对您的数据进行建模
2. **设计系统** — 选择颜色、排版，并设计您的应用外壳结构（application shell）
3. **板块设计（Section Design）** — 针对每个功能区域：指定要求，生成示例数据，并设计屏幕（界面）
4. **导出** — 生成完整的移交包以供后续实现

每一步都是一次对话。AI 会提出问题，您提供方向，然后一起塑造一个符合您愿景的产品——在任何代码实现开始之前。
