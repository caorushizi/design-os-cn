# 使用方法 (Usage)

Design OS 借由斜杠命令（slash commands）指引您贯彻始终的走完一趟设计加工过程。每个命令背后都关联着一段对话沟通回合 —— 您的 AI 工具提出询问，您提供指令导向，随后齐心协力地塑造打磨出您的产品原型。

## 具体的设计工作流 (The Design Workflow)

Design OS 严守着一条经过架构组织的行动顺序列。每一阶段步骤的开展皆建构立足于前一环工作之上。

### 阶段 1: 产品规划 (Product Planning)

不急着画任何界面，先打好这个产品本身内在深处的牢固地基：

1. **产品愿景 (Product Vision)** — 在这股浑然一体无跳脱割裂的对话交谈中去定调敲定您的产品本身，下属囊括的功能主次版块，连同勾画出数据大体的外形结构。
2. **设计通用标记 (Design Tokens)** — 去选取定调属于本工程的主旋律色彩以及其排印字体风格。
3. **程序应用外壳 (Application Shell)** — 规划设计那一层全局包裹着内里各个分页内容的跳转导引菜单以及总体骨骼布局。

关于执行这当中的每一个命令行的细节深层次解说请参见 [产品规划 (Product Planning)](product-planning.md)。

### 阶段 2: 具体功能版块设计 (Section Design)

一经底盘基础构筑落定，即需为路线规划图（roadmap）上被分割开来的那些主体大版块进行逐个作业攻坚：

1. **塑形板块 (Shape the Section)** — 为这版划范围（scope）、立规范需求、同时指令电脑替其批量制造并附赠模拟用到的示例填料数据文本和供TypeScript校验核验规矩准绳用的各类接口规范（types）。
2. **搭建界面呈现屏幕 (Design the Screen)** — 动真刀真枪实战输出构建编写能派去一线战场投入被运行解析执行的那些 React 代码组件实体。
3. **抓取留存视觉凭证实物图 (Capture Screenshots)** —  生成保存对应屏幕截取作为文档留存记录（可选做项）。

给路线规划图里的每部分单独执行这套套路动作直至覆盖完整。

欲去知晓其中各个执行指令（Command）的奥妙玄机解说，移步研习翻阅 [版块细致化设定设计 (Designing Sections)](design-section.md)。

### 阶段 3: 整体打包输出 (Export)

当所有该囊括的独立个体板块皆完成自身独立完整的设计任务时即可执行：

1. **导出输出 (Export)** — 将之前阶段积累缔造而成的成果归拢收集，去整合自动生成出这可以去移交派发出去的齐备交接包围组合（handoff package）。

欲了解当中涉及包含的交付产出有何种种类及应当作何用途来用这些组件物料，请仔细阅读 [包捆移交输出机制详解 (Export)](export.md) 获取使用方法真传。

## 命令快捷索引 (Quick Reference)

| 命令命令行 | 目的用途 |
|---------|---------|
| `/product-vision` | 下定义去诠释描述产品自身概括，确立远景发展大路线规划分支（roadmap sections）与确定关联实体信息的体态面貌框架（data shape） |
| `/design-tokens` | 主张选用属于该体系的色相及采用何等字迹字体 |
| `/design-shell` | 定义设计排版框架布局及其带你跳跃去其它功能角落引路导引系统 |
| `/shape-section` | 当针对具体一块下属业务时为它明确业务边际管辖范围（scope），定义确切的运转规则制度（requirements），及下死命令逼其给出测试伪样本（sample data）伴随对应检查校验规范（types） |
| `/design-screen` | 创建界面屏幕设计的对应 UI 反应组件（React components） |
| `/screenshot-design` | 生成拍摄能够定格瞬间实样成品的静态像素截画图片归集储藏 |
| `/export-product` | 将上面一系列动作产出生成的最终资产文件等进行归口聚拢结伴封箱，打个全副武装的移交发货（handoff）交付产物大包 |
| `/product-roadmap` | 于草创（initial creation）期度过之后，对过往产品划分切割的宏大阶段路线区块目标分布体系作出补修更新调整 |
| `/data-shape` | 当大框架最初阶段起好根脚（initial creation）后，针对数据内在要素彼此挂靠连结羁绊逻辑等去施加以变更 |
| `/sample-data` | 同样是在度越了启始初始开创期之后去改动、变更新替此前早早定好的测试示范模拟假象的临时替补凑数用样本材料与及其配合做类型安全把关用的人防哨兵规则限制（types） |

## 相关锦囊提示 (Tips)

- **严格遵循此行事的次序步骤流水线 (Follow the sequence)** — 这其中的这套流程招式都是步步为营以先前做派举动充作此番底料踏脚石来进行下去的，绝不可以跳出不循序渐进贪图妄图图去抄写捷径而贸然跨越式的急流勇进。
- **指示指令说辞应当具象入微 (Be specific)** — 提供赐予喂出的参数描绘情形内容描述情报供给越是巨细靡遗精粹深刻，便越好，越有效果与回报。
- **允许回炉调适，打磨升级 (Iterate)** — 各单独分开孤立体执行着的任务操作说到底就是交谈聊天磨合沟通的一来二回式的交互动作组合，可以不断的试错来不断调适修正提纯以贴近达致那您内心理所期盼图景的理想美景蓝图。
- **记得重载热起服设开发节点服务器 (Restart the dev server)** — 凡但经历在其中孕育成化作新编写构建生成产出过的功能元件、视图切面组合等以后须牢记，执行一遍复位重置以致那些才加进去新鲜玩意变更的效能变化生效发挥它的直观视觉威力。
